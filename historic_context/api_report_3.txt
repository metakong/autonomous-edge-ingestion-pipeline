For a self-contained "AGI" running on Android in 2025, the strategy changes from "downloading databases" (which kills mobile storage) to High-Velocity Live Retrieval. You need lightweight REST/JSON APIs that allow the AI to "sense" the world in real-time without bloating the APK size.
Here is the exhaustive architecture of Free, Public, Industrial-Strength APIs that act as the sensory organs for a mobile AI.


I. The "Cerebral Cortex" (General Knowledge & Facts)


These APIs provide the raw encyclopedic data to ground the AI's reasoning.


1. Wikidata Query Service (SPARQL)


* Role: The ultimate fact database. Instead of storing "Paris is the capital of France," your AI queries this relationship instantly.
* API Endpoint: https://query.wikidata.org/sparql
* Data: Structured data for ~100 million items (concepts, people, places).
* Mobile Strategy: Do not pull dumps. Send optimized SPARQL queries to extract specific properties (e.g., "Get me the height and birth date of [Entity]").
* Cost/Limit: Free. Public. Complex queries may timeout (60s limit), so code your AI to write efficient SPARQL.


2. Wikipedia REST API (Summary Endpoint)


* Role: Context and prose.
* API Endpoint: https://en.wikipedia.org/api/rest_v1/page/summary/{title}
* Data: Returns the intro paragraph, main image, and basic metadata of any article.
* Why for Android: It saves bandwidth by not downloading the full HTML page, returning just the high-value "summary" JSON.
* Cost/Limit: Free. High volume allowed with a proper User-Agent header.


3. ConceptNet 5.7


* Role: Common Sense Reasoning. (e.g., telling the AI that "To eat, you need food" or "Ice is cold").
* API Endpoint: http://api.conceptnet.io/
* Data: A knowledge graph connecting words by meaning rather than just definition.
* Cost/Limit: Free. Rate-limited (~60 req/min), but sufficient for "thought processes."

II. The "Eyes & Ears" (Real-Time World Access)


True AGI needs to know what is happening right now.


4. Brave Search API (The "Google" Replacement)


* Role: Web Search. The only major search engine with a usable free tier for developers in 2025.
* API Endpoint: https://api.search.brave.com/res/v1/web/search
* Data: Real-time search results, news snippets, and answering questions from the web.
* Free Limit: 2,000 queries/month (Free tier). This is enough for a personal assistant but tight for scaling. It is the only robust option; Google/Bing are paid or extremely limited (100/day).


5. GDELT Project (Global News Intelligence)


* Role: "The Planet's Pulse." A massive database of global society.
* API: https://api.gdeltproject.org/api/v2/doc/doc
* Data: Monitors broadcast, print, and web news from nearly every country in over 100 languages.
* Usage: Your AI can ask, "What is the sentiment in Germany regarding nuclear energy today?" and get a data-driven trend line.
* Cost/Limit: 100% Free and Open.


6. Open-Meteo (Global Weather)


* Role: Environmental awareness.
* API Endpoint: https://api.open-meteo.com/v1/forecast
* Data: Weather, air quality, and marine conditions.
* Why: Unlike others, it requires no API key and is free for non-commercial use.
* Limit: 10,000 requests/day (huge for a single user).
________________


III. The "Visual & Spatial" Cortex (Maps & Media)




7. OpenStreetMap (via Nominatim)


* Role: Geocoding (turning "Milwaukee" into "43.0389° N, 87.9065° W") and reverse geocoding.
* API Endpoint: https://nominatim.openstreetmap.org/search
* Limit: strict 1 request/second. Requires a unique User-Agent.
* Constraint: For heavier use, switch to the Photon API (based on OSM) which is often less restrictive.


8. Wikimedia Commons API


* Role: Image recognition context.
* Data: Access to millions of free-to-use images and media files.
* Usage: If the user asks "Show me a 1967 Mustang," the AI retrieves the file URL from here.
________________


IV. The "Academic Brain" (Science & Math)




9. OpenAlex (Scientific Papers)


* Role: Deep research. Replaces Google Scholar.
* API Endpoint: https://api.openalex.org/
* Data: Metadata for 250M+ scientific works.
* Limit: 100,000 requests/day (Polite pool).
* Use Case: "Find me the latest papers on transformer architecture from 2024-2025."


10. SymPy (Local Python Library - Not an API)


* Crucial Distinction: For an "Android AGI," you do not want to call Wolfram Alpha (Paid).
* Solution: Embed SymPy (Python library) directly into your Android app using Chaquopy or Termux.
* Capability: It allows your AI to solve calculus, algebra, and physics equations locally on the device's CPU without internet. This is superior to any free API.
