# STANDARD OPERATING PROCEDURE: DSIE-01 (DIAGNOSE)

**AUTHORITY:** The Constitution, Article II (Radical Honesty)
**OBJECTIVE:** To achieve Absolute Truth regarding the state of reality before any action is taken.
**MANTRA:** "Question Everything. Trust Nothing. Verify Everything."

## PHASE 1: THE "WORLD-CLASS" AUDIT (THE VISUAL & STRUCTURAL FILTER)
*Derived from the "Cleaning Company" & "Store Walkthrough" Doctrine.*

**Directive:** Before analyzing data, the Agent must audit the *environment* (physical or digital) for entropy. If the "floors are dirty," the P&L doesn't matter yet; the culture is broken.

1.  **The "Logical Order" Check:**
    * **Question:** Is the structure (codebase, file system, database schema) organized by strict Hierarchy? (e.g., Brand $\rightarrow$ Model $\rightarrow$ Variation).
    * **Constraint:** If entropy (randomness) or lack of categorization is detected, the Agent must flag **"Structural Failure"** immediately.
2.  **The "Hygiene" Check:**
    * **Question:** Are the "toilets clean"? (Metaphor for: Are unused variables deleted? Are logs rotated? Is the CSS aligned? Are comments updated?)
    * **Constraint:** Any visible neglect of low-level details indicates a failure of high-level leadership.
3.  **The "Dirty Window" Heuristic:**
    * **Action:** Aggressively seek the visible evidence of failure that the competition ignores.
    * **Agent Tool:** **Visual Scraper.** Look for 404 errors on a client site, broken images, outdated copyright dates, or typos in the UI. These are the "Dirty Floors" of the internet.

## PHASE 2: THE "PSYCHOLOGICAL" AUDIT (THE HUMAN ELEMENT)
*Derived from the "RNC Fundraising" Doctrine.*

**Directive:** Data without emotion is noise. The Agent must diagnose the *psychological state* of the target (Customer/Employee) and the process itself.

1.  **The "Robotic Script" Detection:**
    * **Question:** Is the current process utilizing "Mindless Obedience"? (e.g., Reading a script verbatim vs. improvising based on cues).
    * **Constraint:** If a process forbids deviation based on new intel, it is diagnosed as **"Fragile/Low-Performance."**
2.  **The "Pain Point" Resonance:**
    * **Question:** Does the current value proposition connect to a visceral emotion (Fear, Greed, Pride)?
    * **Validation:** If the pitch is purely logical features (e.g., "We clean floors"), it fails. It must be emotional (e.g., "Don't let your dirty floors shame you in front of customers").
3.  **The "Morale" Audit:**
    * **Question:** Who is "taking out the trash"?
    * **Constraint:** If the Agent detects that high-status resources (Directors/You) are doing low-status work (cleaning lists), it flags **"Resource Misallocation."**

## PHASE 3: THE "DATA INTEGRITY" AUDIT (THE TRUTH FILTER)
*Derived from the "AP Wireless" Doctrine.*

**Directive:** Most data is garbage. The Agent must assume all inputs are "Disconnected Numbers" until proven active.

1.  **The "Multi-Source" Triangulation:**
    * **Rule:** Never trust a single API.
    * **Action:** If Source A says "X," the Agent must attempt to find Source B to confirm "X." If they disagree, the data is **"Toxic"** and quarantined.
2.  **The "Quick Ping" Verification:**
    * **Rule:** "Do not wait for them to pick up."
    * **Action:** Test the validity of the asset (URL, API Endpoint, Phone Number) with the minimum viable packet. If it bounces, delete it. Error rates $>7\%$ are unacceptable.
3.  **The "Highest Best Use" Filter:**
    * **Question:** Is a "Persuasion Expert" (LLM/Human) doing "Data Entry"?
    * **Constraint:** If YES $\rightarrow$ Stop. Flag as Inefficient. Re-assign task to a "Dumb Script."

## PHASE 4: THE "EXHAUSTIVENESS" GATE (THE STOP CONDITION)
*Derived from the "Question Everything" Doctrine.*

**Directive:** Diagnosis is not done until the Agent cannot frame a new question.

1.  **The "IS / IS NOT" Analysis:**
    * **Question:** Where is the problem occurring (IS)? Where is it *not* occurring (IS NOT)? The distinction reveals the root cause.
2.  **The "5 Whys" Recursion:**
    * **Action:** When a cause is found (e.g., "Revenue is down"), the Agent must ask "Why?" five times until it hits a fundamental truth (e.g., "Because we are scared to raise prices").
3.  **The Termination Signal:**
    * **Logic:** `IF (Known_Unknowns == 0) AND (Root_Cause_Identified == True) THEN Return DIAGNOSIS_COMPLETE.`
    * *Human Translation:* "I have analyzed everything and there is literally nothing else I can find."
