To build a system with access to "all known facts," relying solely on standard REST APIs (sending one request per fact) is inefficient and will hit rate limits immediately.
The "Expert" approach used by data engineers in 2025 is to use Hybrid Access: use Bulk Data/Dump APIs to ingest the core "World Knowledge" into your own local database (SQL/Vector), and then use Live Lookup APIs only for real-time updates or specific queries.
Here is the list of free, publicly available APIs that, when combined, cover the majority of human knowledge.


I. The "General Knowledge" Core (Structured Facts)


These are the direct equivalents to Wikipedia but designed for machines.


1. Wikidata (The Backbone)


* What it is: The structured database that powers Wikipedia. It contains nearly 100 million "items" (people, places, concepts) and their connections.
* 2025 Update: As of 2025, Wikidata has split its query service to manage load. You now have two endpoints:
   * Main Graph: query.wikidata.org (General facts, history, geography).
   * Scholarly Graph: query-scholarly.wikidata.org (Scientific papers and citations).
* How to use: Use the SPARQL API for complex queries (e.g., "Give me all cities with a population > 1M situated on a river").
* Expert Tip: For "all facts," do not scrape the API. Use the JSON Dumps (updated weekly) to load the entire graph into a local graph database (like Neo4j or Apache Jena).


2. DBpedia


* What it is: A community project that extracts structured content from the information boxes in Wikipedia.
* Status: The 2025-06 Release is the most current, offering cleaner data consistency than raw Wikidata for certain topics.
* Best For: Semantic queries and connecting different datasets (e.g., linking a MusicBrainz ID to a Wikipedia article).


3. ConceptNet (Common Sense)


* What it is: A knowledge graph of "common sense" rather than encyclopedic facts. (e.g., "An apple is a fruit," "You use a spoon to eat soup").
* API Limit: ~3,600 requests/hour.
* Why you need it: Wikipedia tells you who Napoleon was; ConceptNet tells your AI that "Generals lead armies" and "Armies fight wars."

II. Science & Academia (The "Deep" Knowledge)


General LLMs often hallucinate on specific scientific papers. These APIs provide ground truth.


4. OpenAlex (The New Standard)


* What it is: The fully open successor to Microsoft Academic Graph. It indexes hundreds of millions of scientific papers, authors, and institutions.
* Limits: The "Free" tier allows 100,000 requests per day (extremely generous).
* Data: Returns metadata, abstracts, and citation links.
* Access: They offer a highly efficient API, but for a "local brain," you can download their entire dataset (approx. 300GB compressed) via AWS S3 (requester pays) or their snapshot service.


5. ArXiv API


* What it is: Access to pre-print papers in Physics, CS, Math, and AI.
* Limits: Strict (~1 request every 3 seconds).
* Expert Tip: Use the OAI-PMH (Open Archives Initiative) endpoint for bulk harvesting metadata rather than the standard search API.


6. Crossref


* What it is: The Digital Object Identifier (DOI) registration agency. The ultimate source for verifying if a piece of academic work exists.
* 2025 Update: New rate limits were imposed in late 2025, but the "Polite Pool" (adding your email to the header) still allows for substantial throughput.
________________


III. Cultural & Media Facts




7. Open Library (Internet Archive)


* What it is: A webpage for every book ever published.
* Warning: They explicitly ban using their REST API for bulk scraping.
* The Solution: They provide a Bulk Data API where you can download their entire catalog (Authors, Works, Editions) as a tab-separated file. Use this to build your local library database, then use the Live API only for cover art or availability checks.


8. MusicBrainz


* What it is: The open music encyclopedia.
* Limits: 1 request/second (Live API).
* Expert Tip: MusicBrainz provides a pre-built Virtual Machine (VM) and database dumps. If you are building a serious application, downloading the dump is the only approved way to access "all facts" about music without getting banned.


9. Europeana


* What it is: API access to millions of digitized artworks, books, and artifacts from European museums.
* Best For: Historical / Art History facts that are often missing from US-centric datasets.
________________


IV. Global Statistics (The "State of the World")




10. World Bank Open Data API


* What it is: Time-series data for development indicators (GDP, literacy, health, energy) for every country.
* Why: It provides the raw numbers that back up the facts in Wikipedia.

Summary: The "Fact Stack" Architecture


To achieve your goal of "access to all known facts," you should not view these just as APIs to query, but as sources to replicate.
